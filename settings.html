PingOne Worker App Setup Guide

Step 1: Create a Worker Application

  1. Log into your PingOne Admin Console
  2. Navigate to Applications ‚Üí Applications
  3. Click "+ Add Application"
  4. Select "Worker Application"
  5. Give it a descriptive name (e.g., "User Import Worker")
  6. Click "Save"

Step 2: Configure Application Settings

  1. In your worker application, go to "Configuration"
  2. Copy the Client ID and Client Secret
  3. Note your Environment ID (found in the URL or environment settings)
  4. Ensure the application is enabled

Step 3: Grant Required Permissions

Required Scopes:

  • p1:admin:user:read - Read user information
  • p1:admin:user:write - Create and modify users

Step 4: Verify Your Credentials

Important: Double-check that you've copied the credentials exactly as shown in PingOne. Even a single character difference will cause authentication failures.

Common Issues & Solutions

‚ùå "Invalid client" error:

  • Verify the Client ID is correct
  • Ensure the worker application is enabled
  • Check that you're using the right environment

‚ùå "Invalid client_secret" error:

  • Copy the Client Secret exactly as shown
  • Don't include any extra spaces or characters
  • Regenerate the secret if needed

‚ùå "Environment not found" error:

  • Verify the Environment ID is correct
  • Ensure you're using the right environment
  • Check the environment is active
